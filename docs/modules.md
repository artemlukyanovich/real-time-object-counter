# Описание модулей

---

## `src/main.py` — Оркестратор

**Ответственность:** точка входа приложения, координация всех компонентов pipeline, главный цикл обработки кадров.

**Основные классы:**

### `ObjectCounterApp`

| Метод | Описание |
|-------|----------|
| `__init__(config_path, source_override)` | Загружает конфиг, создаёт все компоненты |
| `_initialize_components()` | Инстанциирует VideoSource, Detector, Tracker, Counter, Renderer, Metrics |
| `run()` | Главный цикл: читает кадр → обрабатывает → отображает |
| `_render_frame(frame, detections, tracked_objects, counts, fps)` | Условная отрисовка по флагам конфига |
| `cleanup()` | Освобождение ресурсов, вывод итоговых метрик |

**Входные данные:** `config_path: str`, `source: int | str` (из CLI)

**Выходные данные:** отображение в окне OpenCV, печать итоговых метрик в stdout

**Роль в pipeline:** единственный модуль, который знает обо всех остальных. Передаёт данные между компонентами последовательно в каждой итерации цикла.

## `src/video_source.py` — Источник видео

**Ответственность:** захват кадров из веб-камеры или видеофайла, нормализация разрешения.

**Основные классы:**

### `VideoSource`

| Метод | Описание |
|-------|----------|
| `__init__(source, width, height)` | Открывает cv2.VideoCapture, устанавливает разрешение |
| `read()` | Возвращает `(bool, frame)`, ресайзит до заданного разрешения |
| `release()` | Закрывает захват |
| `get_fps()` | FPS источника (из метаданных видео или камеры) |
| `get_frame_count()` | Текущий номер кадра |

**Входные данные:**
- `source`: `int` (индекс камеры) или `str` (путь к файлу)
- `width`, `height`: целевое разрешение

**Выходные данные:** `Tuple[bool, Optional[np.ndarray]]` — флаг успеха и кадр в формате BGR

**Роль в pipeline:** первое звено. Поставляет сырые кадры для детектора.

## `src/detector.py` — Детектор объектов

**Ответственность:** инференс YOLOv8 на кадре, конвертация сырого вывода модели в структурированный формат.

**Основные классы:**

### `ObjectDetector`

| Метод | Описание |
|-------|----------|
| `__init__(model_name, confidence_threshold, device)` | Загружает веса YOLO, настраивает устройство |
| `detect(frame)` | Запускает инференс, возвращает список детекций |
| `get_class_names()` | Словарь `{id: class_name}` из модели |

**Входные данные:** `frame: np.ndarray` (BGR, HxWx3)

**Выходные данные:**
```python
List[Tuple[Tuple[int, int, int, int], str, float]]
# [((x1, y1, x2, y2), class_name, confidence), ...]
```

**Роль в pipeline:** второе звено. Преобразует сырой кадр в список обнаруженных объектов с координатами и классом.

**Примечание:** использует `box.xyxy[0].int().tolist()` — абсолютные пиксельные координаты в формате `(left, top, right, bottom)`.

## `src/tracker.py` — Трекер объектов

**Ответственность:** сопоставление детекций между кадрами, присвоение и поддержка уникальных ID объектов.

**Основные классы:**

### `CentroidTracker`

| Метод | Описание |
|-------|----------|
| `__init__(max_disappeared, max_distance)` | Инициализирует состояние трекера |
| `update(detections)` | Сопоставляет новые детекции с существующими треками |
| `get_tracks()` | Текущие позиции треков `{id: (cx, cy)}` |
| `reset()` | Сброс всего состояния |

**Алгоритм `update`:**
1. Вычислить центроид каждой детекции
2. Для каждого существующего трека найти ближайшую детекцию (евклидово расстояние)
3. Если расстояние < `max_distance` — обновить трек
4. Непривязанные детекции → новые треки с инкрементным ID
5. Треки без совпадений → инкремент `disappeared`; при `disappeared > max_disappeared` — удалить

**Входные данные:** `List[((x1,y1,x2,y2), class_name, confidence)]`

**Выходные данные:**
```python
Dict[int, Tuple[Tuple[int, int, int, int], str]]
# {track_id: ((x1, y1, x2, y2), class_name)}
```

**Роль в pipeline:** третье звено. Превращает независимые детекции в непрерывные треки объектов.

**Архитектурное ограничение:** нет обработки окклюзий — объекты, перекрывающие друг друга, могут потерять ID.

## `src/counter.py` — Счётчик объектов

**Ответственность:** учёт уникальных объектов, опциональная фильтрация по зоне подсчёта.

**Основные классы:**

### `ObjectCounter`

| Метод | Описание |
|-------|----------|
| `__init__(count_zone=None)` | Инициализирует счётчики; `count_zone` — полигон `List[Tuple[int, int]]` |
| `update(tracked_objects)` | Обновляет счётчики по текущим трекам |
| `get_counts()` | `{class_name: count}` |
| `get_total_count()` | Суммарное количество |
| `get_class_count(class_name)` | Счётчик конкретного класса |
| `set_count_zone(zone)` | Динамически задать зону подсчёта |
| `reset()` | Сброс всех счётчиков |

**Логика подсчёта:** каждый `track_id` попадает в `counted_ids` ровно один раз. Повторное появление в кадре — не считается. Если задана `count_zone`, объект считается только при нахождении его центроида внутри полигона.

**Входные данные:** `Dict[int, Tuple[Tuple[int,int,int,int], str]]`

**Выходные данные:** `Dict[str, int]` — `{"person": 5, "car": 2}`

**Роль в pipeline:** четвёртое звено. Агрегирует статистику объектов по классам.

## `src/renderer.py` — Рендерер

**Ответственность:** визуализация всех данных pipeline поверх кадра.

**Основные классы:**

### `FrameRenderer`

| Метод | Описание |
|-------|----------|
| `__init__(font_size=0.7)` | Инициализирует параметры шрифта, генерирует 256 цветов |
| `render_detections(frame, detections, show_confidence)` | Рисует bbox детекций |
| `render_tracks(frame, tracked_objects, show_ids)` | Рисует bbox треков с ID |
| `render_counts(frame, counts, total_count)` | Полупрозрачная панель со счётчиками |
| `render_fps(frame, fps)` | FPS в правом верхнем углу |
| `_draw_label(frame, text, x, y, bg_color)` | Подпись с фоном |
| `_generate_colors(n=256)` | HSV → BGR, 256 цветов |

**Входные данные:** `np.ndarray` + данные от предыдущих модулей

**Выходные данные:** `np.ndarray` с аннотациями (тот же буфер, модифицированный in-place)

**Роль в pipeline:** финальное звено перед отображением. Не влияет на логику обработки.

**Цветовая схема:**
- Белый текст на цветном фоне (bbox треков)
- Зелёный текст для метрик FPS
- Полупрозрачный чёрный фон для панели счётчиков

## `src/metrics.py` — Метрики производительности

**Ответственность:** измерение и агрегация временны́х характеристик pipeline.

**Основные классы:**

### `PerformanceMetrics`

| Метод | Описание |
|-------|----------|
| `__init__(window_size=30)` | Создаёт deque-буферы для скользящего окна |
| `record_frame_time(elapsed)` | Записывает время обработки кадра (сек) |
| `record_detection_time(elapsed)` | Записывает время инференса (сек) |
| `record_tracking_time(elapsed)` | Записывает время трекинга (сек) |
| `get_fps()` | `1.0 / avg_frame_time` по последним 30 кадрам |
| `get_average_detection_time()` | Среднее время детекции (мс) |
| `get_average_tracking_time()` | Среднее время трекинга (мс) |
| `get_summary()` | Сводный словарь всех метрик |
| `reset()` | Очистка буферов |

**Входные данные:** `float` (время в секундах)

**Выходные данные:** `Dict[str, float]` — fps, avg_detection_time_ms, avg_tracking_time_ms, total_frames, elapsed_seconds

**Роль в pipeline:** сквозной мониторинг, не встроен в цепочку обработки данных.

## `src/utils.py` — Утилиты

**Ответственность:** геометрические вычисления, используемые трекером и счётчиком.

| Функция | Описание |
|---------|----------|
| `get_centroid(bbox)` | `(x1,y1,x2,y2)` → `(cx, cy)` |
| `distance(p1, p2)` | Евклидово расстояние между двумя точками |
| `is_point_in_polygon(point, polygon)` | Ray casting, `True` если точка внутри полигона |
| `calculate_iou(box1, box2)` | IoU двух bbox (не используется в текущем pipeline) |

**Примечание:** `calculate_iou` реализован, но не вызывается ни из одного модуля — потенциал для будущего IoU-трекера.
